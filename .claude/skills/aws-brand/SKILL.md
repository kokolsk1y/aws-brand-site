# AWS Brand Site — Brand Skill

Single source of truth для визуала и стиля сайта **AWS Products** (российский бренд электрики, выключатели/розетки/удлинители). Референс — werkel.ru: премиум, тёплый минимализм, медленные анимации, AI-фото с реальными интерьерами.

Используй эти правила при любом редактировании стилей, layout-компонентов, hero, карточек серий, продуктовых страниц, текстов.

---

## 1. Палитра (только эти токены)

```css
/* Базовая шкала */
--black:     #111111
--dark:      #1a1a1a
--gray-800:  #333333
--gray-600:  #666666
--gray-400:  #6b6b6b   /* подписи, label, второстепенный текст */
--gray-200:  #d4d4d4   /* разделители на тёмном */
--gray-100:  #f5f5f3   /* разделители на светлом */
--white:     #ffffff

/* Тёплая основа (werkel-стиль) */
--warm-100:  #f7f4ef   /* основной светлый фон секций */
--warm-200:  #ede8df   /* фон контрастных блоков на светлом */

/* Брендовые акценты */
--warm-gold: #5E8C3A   /* deep green — для светлого фона (CTA, акцент) */
--accent:    #8CB86B   /* AWS green — для тёмного фона */
```

**Правила:**
- Никогда не вводи новые цвета вне этих токенов. Если нужен оттенок — это `rgba()` от существующих с прозрачностью.
- Не используй чисто-белый `#fff` для больших фонов секций — бери `--warm-100`. Чисто-белый только для контраста (карточки на тёплом, glass-morphism).
- CTA на тёмном — `--accent` (#8CB86B). CTA на светлом — `--warm-gold` (#5E8C3A). НЕ путать местами — на светлом #8CB86B бледнеет.
- Текст по умолчанию: `--black` на светлом, `--white` на тёмном. Серая палитра — только для подписей и второстепенного.

## 2. Типографика

- Единственный шрифт: **Montserrat** (400/700/800), уже подключён через Google Fonts в `BaseLayout.astro` и `index.html`.
- `--font: 'Montserrat', -apple-system, BlinkMacSystemFont, sans-serif`
- Никаких других шрифтов. Не добавляй serif/display/handwriting — werkel-стиль строго один sans.
- Размеры (де-факто из текущего CSS):
  - h1 / hero-title: clamp 48–96px, font-weight 200–300 (тонкие!), line-height 1.05
  - h2 / section-title: 40–56px, weight 300
  - body: 16–18px, weight 400
  - labels: 11px, weight 600, letter-spacing 4px, uppercase, цвет `--gray-400`
- **Тонкие веса для крупного, жирные только для уплотнённых caps-меток.** Hero-заголовок в 800-весе сразу убивает «премиум».

## 3. Архитектура CSS — критично

В проекте **два живых CSS-файла**, оба нужно править одновременно при общих изменениях:

| Файл | Что обслуживает | Где импортируется |
|---|---|---|
| `public/style.css` (3900 строк) | Главная страница `index.html` | прямой `<link>` в `index.html` |
| `src/styles/main.css` (2050 строк) | Astro-страницы (`/series/[slug]`, `/category/[slug]`, `/product/[article]`, 404) | `import '../styles/main.css'` в `BaseLayout.astro` |

**Правило:** если меняешь брендовый компонент (header, footer, palette, font, общая кнопка, карточка серии) — правь **оба файла**. Если правишь только Astro-страницу или только главную — один.

Перед редактированием CSS — проверь, не дублируется ли селектор в обоих файлах через Grep. Если дублируется — синхронизируй.

Никогда не создавай третий CSS-файл «для нового раздела». Расширяй существующие.

## 4. Структура секций главной (паттерн чередования)

Главная — лента секций со строгим чередованием тем хедера и фона:

```
Hero (слайдер 5 кадров, fade)            — тема меняется по слайду
О бренде                                 — светлый
Серии (UNO / DESIGN / AURA + конструктор) — светлый
Категории (6 карточек grid 3×2)          — тёмный
Преимущества (4 свайпер-слайда)          — светлый
Где купить (Ozon / Wildberries / СТВ39)  — тёмный
Footer                                   — тёмный
```

Заголовок секции всегда сопровождается **label** (короткая caps-метка над заголовком). Нет label — секция выглядит «голой».

## 5. Анимации

В проекте уже подключены: **GSAP + ScrollTrigger**, **Lenis** (плавный скролл), **Swiper**. **Не добавляй** новые анимационные библиотеки (особенно Framer Motion — это React, у нас Astro+vanilla, конфликт по архитектуре и весу бандла).

**Стандартные паттерны:**
- Fade-up при появлении в viewport (через ScrollTrigger).
- Hover на карточках: лёгкий `scale(1.02)` или `translateY(-4px)`, `transition: 0.4s cubic-bezier(0.23, 1, 0.32, 1)`.
- Hero-слайды — Swiper fade, авто-смена 6 секунд.
- Reveal-анимация подъёма у конструктора серий **была отключена** (создавала щель при возврате) — не возвращай.
- Длительности: 0.3s для мелких hover, 0.4s для тем хедера, 0.45s для side-menu, 0.6–0.8s для scroll-reveal.

## 6. Продуктовые изображения — обязательные правила

PNG товаров серий **Uno** и **Aura** лежат в `public/img/series/` и в продуктовых JSON.

- Все товары серий должны быть в **анфас-ракурсе** (фронтальный вид), не в перспективе и не в разборе. Разобранный выключатель — только для hero exploded-view, нигде больше.
- Серия **Uno SoftTouch**: матовое soft-touch покрытие, бевель по краю. При генерации/замене визуала используй существующие PNG как Subject Reference, не пересоздавай с нуля.
- В карточках товаров `max-width: 100%` и `max-height: 100%` — товары должны быть **крупными**, занимать карточку.
- Рамки выключателей в карточках — **прозрачный фон** между внешней и внутренней рамкой (видна подложка).
- Не применяй фильтр (blur/grayscale/opacity) к карточкам товаров — это «тусклит» серию.

## 7. Тон и копирайт

- Только русский язык. Английский — только в названиях серий (UNO, AURA, DESIGN), технических артикулах и брендовом «AWS Products».
- Тон — спокойный, премиум, без восклицательных знаков и маркетингового пафоса. «Электрика нового стандарта», «внимание к деталям». Не «революция», не «лучший на рынке».
- Слоганы — короткие, 2–4 слова. Длинные описания — только в блоках «О бренде» и продуктовых страницах.
- Цены без копеек, формат «от 173 ₽» (₽, не «руб»).

## 8. Интеграции и MCP

- `21st.dev` MCP может использоваться для **референса блоков** (структура, идеи компоновки), но **не как источник готовых компонентов** в чистом виде — стиль 21st.dev глянцевый/SaaS, не werkel. Если берёшь блок — переделывай под нашу палитру и типографику (см. §1, §2).
- Bunny CDN, Cloudflare и сторонние animation libs — **только с моего согласия**. Сейчас всё на vanilla + GSAP + Swiper, этого хватает.

## 9. Деплой

- Push в `main` = автодеплой через GitHub Actions (Astro build → GitHub Pages).
- Перед коммитом крупных стилевых изменений — проверь визуал в обоих местах: главная (`index.html`) и хотя бы одна Astro-страница (например, `/series/uno`).
- Не ломай `BASE_URL` — все внутренние ссылки идут через `${BASE}/...` или `import.meta.env.BASE_URL`, никаких абсолютных `/img/...` без префикса.

## 10. Что НЕ делать

- Не добавляй Tailwind, Bootstrap, MUI, shadcn — у нас vanilla CSS с токенами.
- Не подключай Framer Motion, GSAP уже есть.
- Не вводи новые шрифты сверх Montserrat.
- Не используй чёрный фон `#000` — только `--black` (#111) или `--dark` (#1a1a1a). Чистый чёрный делает картинку «дешёвой» в этой палитре.
- Не делай резких быстрых анимаций (<200ms) на крупных элементах — werkel-стиль про медленность и инерцию.
- Не создавай `_v2`, `_old`, `_backup` копии файлов — редактируй оригинал.
- Не комментируй код по принципу «что делает» — имена и так читаются. Только «почему» для неочевидных решений.
